************Présente MySQL et MongoDB ************************


MySQL et MongoDB sont deux des fournisseurs de bases de données 
les plus appréciés et les plus compétitifs pour les applications en ligne.
Bien qu’ils fonctionnent tous deux comme des systèmes de gestion de bases de
données (SGBD), qui vous permettent de collecter des données à partir d’un site
Web ou d’une application et de fournir des rapports, ils ne sont pas identiques. 
MySQL est un système obsolète structuré en tables, tandis que MongoDB est 
un système basé sur des documents.

MongoDB est une application de base de données open source multiplateforme 
orientée document et librement accessible. En tant que base de données NoSQL ,
MongoDB utilise des documents sans schéma ressemblant à JSON pour la représentation
des données. Il offre un cadre robuste pour le stockage et la récupération
des données, offrant une approche flexible et évolutive de la gestion des bases 
de données.

SQL est un système de gestion de base de données relationnelle est SQL Server 
(SGBDR). Il est également parfois appelé Microsoft SQL Server ou MSSQL.
Microsoft a créé SQL Server, qui a été mis à disposition pour la première fois 
le 24 avril 1989.
Les données sont organisées en lignes et colonnes dans des tableaux. 
Chaque ligne du tableau représente une entrée ou un enregistrement,
tandis que chaque colonne représente un certain champ ou fonctionnalité.
Les éditions suivantes de Microsoft SQL Server sont disponibles, chacune avec
un ensemble différent de fonctionnalités : Express, Enterprise, Standard, 
Web et Developer.
L'édition Express est un logiciel gratuit pour les applications modestes et 
d'entrée de gamme, mais d'autres éditions, basées sur une licence et utilisées
pour des projets plus importants à des niveaux commerciaux en fonction du service 
requis, ne sont disponibles qu'avec un abonnement payant.

-Chaque enregistrement dans MongoDB est conservé comme un document distinct.

-dans MongoDB les documents d'une classe ou d'un groupe spécifique sont conservés 
dans une « collection » mais a SQL Les lignes (également appelées enregistrements)

-MongoDB prend en charge la réplication et le partitionnement prêts à l'emploi et
